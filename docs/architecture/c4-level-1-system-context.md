# C4 Level 1 — System Context  
## Проект: Букашки сайт

## 1. Назначение документа

Этот документ фиксирует контекст системы **«Букашки сайт»** на уровне **C4 Level 1 (System Context)**.

Цель этого уровня:
- определить назначение системы
- зафиксировать основных пользователей
- обозначить границы системы
- выделить внешние системы, с которыми она взаимодействует

На этом уровне **не рассматриваются**:
- внутренние модули приложения
- классы
- сущности БД
- технические детали реализации

---

## 2. Краткое описание системы

**Букашки сайт** — это веб-система для ведения цифровой коллекции экземпляров насекомых.

Система позволяет пользователям:
- хранить сведения о своих экземплярах насекомых
- прикладывать фотографии и дополнительные данные
- просматривать коллекции других пользователей
- управлять видимостью своей коллекции
- добавлять других пользователей в друзья
- инициировать и проводить обмен экземплярами

---

## 3. Основной пользователь

### Коллекционер насекомых / биолог
Основной пользователь системы — человек, который собирает насекомых и хочет вести цифровую коллекцию своих экземпляров.

Пользователь использует систему, чтобы:
- регистрироваться и входить в аккаунт
- добавлять экземпляры насекомых в свою коллекцию
- редактировать и удалять свои записи
- загружать фотографии экземпляров
- просматривать доступные ему коллекции других пользователей
- управлять приватностью своей коллекции
- отправлять запросы в друзья и принимать их
- создавать предложения обмена экземплярами
- отслеживать состояние обменов

---

## 4. Границы системы

Внутри системы находятся следующие ключевые возможности:
- учет пользователей
- управление коллекцией экземпляров насекомых
- хранение метаданных об экземплярах
- работа с видимостью коллекции
- социальные связи между пользователями
- процесс обмена экземплярами
- просмотр чужих коллекций с учетом прав доступа

Система отвечает за бизнес-логику, связанную с коллекциями, доступом и обменами.

---

## 5. Внешние системы

Ниже перечислены внешние системы, которые могут использоваться системой **«Букашки сайт»**.

### 5.1 Email Service
Внешний почтовый сервис, используемый для:
- подтверждения регистрации
- восстановления доступа
- отправки уведомлений, связанных с обменами и социальными действиями

### 5.2 Image Storage
Внешнее или отдельно выделенное хранилище изображений, используемое для хранения фотографий экземпляров насекомых.

### 5.3 External Species Catalog *(опционально)*
Внешний справочник видов насекомых, который может использоваться для получения или сверки справочной информации о видах.

На первой версии системы эта интеграция не обязательна.

### 5.4 Map / Geocoding Service *(опционально)*
Внешний картографический или геокодинговый сервис, который может использоваться для:
- отображения мест находки на карте
- уточнения географических данных
- преобразования координат и текстовых адресов

На первой версии системы эта интеграция не обязательна.

---

## 6. Ключевые отношения

### Коллекционер насекомых / биолог → Букашки сайт
Использует систему для ведения цифровой коллекции насекомых, просмотра коллекций других пользователей, управления доступом и обмена экземплярами.

### Букашки сайт → Email Service
Использует почтовый сервис для отправки писем, связанных с аккаунтом и уведомлениями.

### Букашки сайт → Image Storage
Использует хранилище изображений для сохранения и получения фотографий экземпляров.

### Букашки сайт → External Species Catalog *(optional)*
Может использовать внешний справочник видов для получения справочной информации.

### Букашки сайт → Map / Geocoding Service *(optional)*
Может использовать картографический сервис для работы с местами находки экземпляров.

## 7. Основные продуктовые решения, влияющие на контекст

На текущем этапе зафиксированы следующие решения:

### 7.1 Объект коллекции — экземпляр, а не просто вид
Система ориентирована на хранение **конкретных экземпляров насекомых**, а не только абстрактного списка видов.

Это позволяет:
- хранить фотографии
- фиксировать метаданные экземпляра
- учитывать происхождение и место находки
- реализовать осмысленный обмен между пользователями

### 7.2 Фотографии являются важной частью модели
Экземпляры насекомых могут сопровождаться фотографиями.

### 7.3 Обмен — это процесс
Обмен рассматривается не как разовое действие, а как отдельный пользовательский процесс.

### 7.4 Видимость коллекции управляется пользователем
Пользователь может определять, кому доступна его коллекция.

Как рабочая модель приватности рассматриваются режимы:
- `PRIVATE`
- `FRIENDS_ONLY`
- `PUBLIC`

### 7.5 Социальные связи являются частью продукта
В системе предполагается поддержка дружбы или аналогичных пользовательских связей для управления доступом и взаимодействием между коллекционерами.

---

## 8. Что не входит в текущую фиксацию контекста

Пока не считаются обязательной частью первой версии:
- чат между пользователями
- real-time уведомления
- аукционы или маркетплейс
- сложная модерация
- развитая ролевая модель
- сложная научная таксономия
- обязательные интеграции со сторонними биологическими базами

Эти возможности могут быть добавлены позже, но не определяют текущий контекст системы.
